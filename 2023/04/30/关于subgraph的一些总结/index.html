Graph简介

Graph是一个索引协议，用于查询以太坊和IPFS等网络。任何人都可以建立和发布开放的API，称为子图，使数据易于访问。子图可以组成一个世界上所有公共信息的全球图。这些数据可以被转化、组织，并在不同的应用程序中分享，任何人只需按几个键就可以查询。

其中主要有四种角色，分别是开发者、索引人、策展人、委托人

开发者

在去中心化应用中创建子图或使用现有子图

索引人

操作节点以索引数据并提供查询

策展人

通过在子图上发出信号来组织数据

委托人

通过将GRT委托给索引人来保护网络

subgraph（子图）

子图可以组成一个世界上所有公共信息的全球图。这些数据可以被转化、组织，并在不同的应用程序中分享，任何人只需按几个键就可以查询。下面我们用一个简单的例子来介绍。

追踪的合约地址：0x9401518f4EBBA857BAA879D9f76E1Cc8b31ed197（mainnet）

子图地址：https://thegraph.com/studio/subgraph/weirdoghost

子图定义由几个文件组成：

  • subgraph.yaml: 包含子图清单的 YAML 文件

    • dataSources.source: the address of the smart contract the subgraph sources, and the ABI of the smart contract to use. The address is optional; omitting it allows to index matching events from all contracts.
    • dataSources.source.startBlock: the optional number of the block that the data source starts indexing from. In most cases, we suggest using the block in which the contract was created.
    • dataSources.mapping.entities: the entities that the data source writes to the store. The schema for each entity is defined in the schema.graphql file.
    • dataSources.mapping.abis：源合约以及您在映射中与之交互的任何其他智能合约的一个或多个命名 ABI 文件。
    • dataSources.mapping.eventHandlers：列出此子图响应的智能合约事件，映射中的处理程序—示例中为./src/mapping.ts—也将这些事件转换为存储中的实体。
    • dataSources.mapping.callHandlers：列出此子图响应的智能合约函数以及映射中的处理程序，该映射将输入和输出转换为函数调用到存储中的实体。
    • dataSources.mapping.blockHandlers: lists the blocks this subgraph reacts to and handlers in the mapping to run when a block is appended to the chain. Without a filter, the block handler will be run every block. An optional call-filter can be provided by adding a filter field with kind: call to the handler. This will only run the handler if the block contains at least one call to the data source contract.

  • schema.graphql: 一个 GraphQL 模式文件，它定义了为您的子图存储哪些数据，以及如何通过 GraphQL 查询这些数据

    zai

    在定义实体之前，重要的是要退后一步，思考数据的结构和链接方式。 所有查询都将针对子图模式中定义的数据模型和子图索引的实体进行。 因此，最好以符合 dapp 需求的方式定义子图模式。 将实体想象为“包含数据的对象”，而不是事件或函数，可能很有用。在例子中我们定义了两种实体，token和user。token的属性有id，tokenId,owner,tokenURI,user的属性有id,balances,tokens，其中带！的属性代表entity创立时该属性不能为空。通过@derivedfrom，我们将token的owner和user的tokens进行了一个多对一映射。

  • AssemblyScript映射: 将事件数据转换为模式中定义的实体(例如本教程中的weirdo-ghost-gang.ts)的 AssemblyScript 代码

映射从特定的源获取数据，并将其转换为在模式中定义的实体。映射是在TypeScript的一个子集中编写的，称为AssemblyScript，可以编译为WASM（WebAssembly）。AssemblyScript比普通TypeScript更严格，但提供了熟悉的语法。
对于子图中定义的每个事件处理程序。映射下的yaml。eventHandlers创建一个同名的导出函数。每个处理程序都必须接受一个名为event的参数，该参数的类型与正在处理的事件的名称相对应。

在例子中我们完成了TransferEvent事件的handler，每当转账发生，我们读取token实体，如果没有则创建新token实体。同理对user实体进行处理，当需要调用合约时，我们将合约地址与schema中生成的合约工厂链接，给user的balances属性进行赋值。note:对于实体的读取必须有是否为空的判断，同时生成完一定要使用save进行保存哦。
